I'm not sure I understand your question. But one of the active periods that is always kept track of is rotation 0, so you can't for example place 02 because the canonical mapping of period 0 would be 01.

The canonical representations are not uniformly distributed, there are many more starting with 00 than starting with 01. Because the string 01 can never contain a repeated character. Does that matter?

I wrote a simple minimal implementation that has some O(n) operations that I'm sure you'll happily ignore because a fancy 'global' data structure could get rid of them (as in my original post). The main thing is that the recurrence relationship of the active periods is laid bare hopefully. https://pastebin.com/H9bkJMfV

I think that once you can prove that the amount of active periods does not grow with N or K the rest of the algorithm is almost trivially CAT (it transparently does not do N or K computations per added symbol, you only have to prove that all partial strings of N-1 have the proper relationship to strings of length N, but this is in the literature.)

Active periods are my own invention.

An active period is any relevant transformation of the current string so that t(s) = s. Which for normal necklaces is just rotations: 001001001 has an an active period of 3, regardless of whether the string is finished.

​

For unlabelled necklaces there might be many more such periodicities as only the 'shape' of the string has to match in any rotation. So for example the partial string 01012..... is so far periodic in rotation [0, 3, 4] (eg: 01012, 12, 1 all match the head of 01012). In fact in this pastebin are all the periodicities in all the strings that are build with the algorithm for N=7 K=3.
